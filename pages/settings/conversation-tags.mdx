## What are Conversation Tags?

**Conversation Tags** (located at **Settings > Conversation Tags**) are labels that can be applied to individual conversations to categorize and organize them.

Tags make it easy to filter the Conversations Inbox, generate targeted reports, and trigger specific workflows based on the nature of an interaction.

## Why Use Tags?

Tags are a simple but powerful organizational tool. Here are some common ways teams use them:

- **Issue Categorization**: Label conversations by the type of issue (e.g., "Billing Issue", "Technical Bug", "Feature Request", "Account Question").
- **Priority Levels**: Mark urgent conversations (e.g., "High Priority", "VIP Customer", "Escalated").
- **Sales Stage**: Track where a lead is in the pipeline (e.g., "New Lead", "Demo Scheduled", "Proposal Sent", "Closed Won").
- **Channel Source**: Identify where a customer originally came from (e.g., "From Website", "From Social Ad", "Referral").

## Creating a Tag

To create a new conversation tag:

1. Navigate to **Settings > Conversation Tags**.
2. Click **Create Tag**.
3. Enter a **Tag Name** (e.g., "Billing Issue").
4. Select a **Color** for the tag. Colors help agents quickly identify tags in the conversation list.
5. Optionally, add a **Description** to clarify when the tag should be used.
6. Click **Save**.

## Applying Tags to Conversations

Tags can be applied to conversations in several ways:

1. **Manually by an Agent**: In the Conversation Detail view, agents can click the **Tags** area in the header and select the appropriate tags from a dropdown list.
2. **Automatically by AI**: You can configure your AI assistants to automatically apply specific tags based on the content of a conversation (e.g., if the user mentions "refund", automatically apply the "Billing Issue" tag).
3. **Via Routing Rules**: When a Routing Rule fires, it can apply a tag as part of its action (e.g., when a conversation is escalated to the Tier 2 team, automatically tag it as "Escalated").

## Using Tags for Filtering and Reporting

Once tags are applied to conversations, they become a powerful filtering tool.

- **In the Inbox**: Use the tag filter in the Conversations Inbox to view only conversations with a specific label (e.g., "Show me all open 'Billing Issue' conversations").
- **In Analytics**: The Analytics dashboard allows you to filter reports by tag, so you can see metrics like resolution time or handoff rate for a specific category of issue.

## Managing Tags

As your categorization needs evolve, you can update your tag library.

- **Edit**: Change the name or color of an existing tag. The update will be reflected on all conversations that already have the tag applied.
- **Delete**: Permanently remove a tag. It will be removed from all conversations it was previously applied to.

<Callout kind="warning">
**Deleting a tag is irreversible.** All historical data associated with that tag will be lost. If you want to retire a tag without losing the data, consider renaming it to something like "DEPRECATED - Billing Issue" instead of deleting it.
</Callout>
