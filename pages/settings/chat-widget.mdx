## What is the Chat Widget?

The **Chat Widget** (configured at **Settings > Chat Widget**) is the customer-facing interface that lives on your website. It allows visitors to interact with your AI assistants or human agents in real-time.

From this settings page, you can customize how the widget looks, how it behaves, and get the code snippet needed to install it on your site.

## Installing the Widget

To add the Autoch.at widget to your website, you need to embed a small piece of JavaScript code.

1. Navigate to **Settings > Chat Widget**.
2. Locate the **Installation Code** section.
3. Click **Copy Code**.
4. Paste this code snippet into the `<head>` or just before the closing `</body>` tag of your website's HTML.

If you use a Content Management System (CMS) like WordPress, Shopify, or Webflow, you can usually add this code through their respective theme settings or by using a custom code plugin.

## Customizing Appearance

While the core colors and logos are pulled from your [Branding Settings](/pages/settings/branding), you can further customize the widget's appearance here.

- **Widget Position**: Choose whether the chat launcher button appears on the bottom-left or bottom-right of the screen.
- **Launcher Icon**: Select the icon that appears on the closed widget button (e.g., a chat bubble, a question mark, or your company logo).
- **Header Text**: Customize the title displayed at the top of the open chat window (e.g., "Chat with Us", "Support", or "Autoch.at Assistant").

## Configuring Behavior

You can control how and when the widget interacts with your website visitors.

### Initial Greeting

The **Initial Greeting** is the first message a visitor sees when they open the chat widget.

- **Welcome Message**: Set a friendly, inviting message (e.g., "Hi there! How can we help you today?").
- **Suggested Prompts**: Provide 2-3 clickable options to help users start the conversation quickly (e.g., "I need technical support", "I have a billing question", "I want to book a demo"). These prompts can be tied directly to specific [Playbooks](/pages/ai-config/playbooks).

### Launch Rules (Proactive Chat)

Instead of waiting for the user to click the widget, you can configure **Launch Rules** to proactively open the chat window based on specific triggers.

- **Time on Page**: Open the widget after the user has been on the page for a certain number of seconds (e.g., 30 seconds).
- **Scroll Depth**: Open the widget when the user scrolls past a certain percentage of the page (e.g., 50%).
- **Exit Intent**: Open the widget when the user's mouse cursor moves towards the top of the browser window, indicating they are about to leave the site.

<Callout kind="tip">
**Use proactive chat sparingly.** While it can increase engagement, popping up the chat window immediately on every page load can be annoying. Target specific high-value pages (like pricing or checkout) with relevant, contextual greetings.
</Callout>

## Advanced Settings

For developers, the Chat Widget settings page also provides access to advanced configuration options.

- **Allowed Domains**: For security, you can restrict the widget to only load on specific domains (e.g., `yourwebsite.com`). This prevents unauthorized use of your widget code on other sites.
- **JavaScript API**: Access documentation for the Autoch.at JavaScript API, which allows you to programmatically open/close the widget, pass custom user data (like a logged-in user's ID or email), or trigger specific events from your website's code.
