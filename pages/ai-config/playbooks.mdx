## What is a Playbook?

While an Assistant's persona defines *how* it speaks (its tone and personality), a **Playbook** defines *what* it should achieve in a conversation.

Playbooks are structured, goal-oriented instructions that guide an AI assistant through a specific process. They are ideal for repeatable workflows like:
- Qualifying a sales lead (BANT: Budget, Authority, Need, Timeline)
- Troubleshooting a technical issue step-by-step
- Collecting information for a support ticket
- Booking an appointment or scheduling a demo

## Creating a Playbook

To create a new playbook, navigate to **AI Assistant > AI Playbooks** and click **Create Playbook**.

### Playbook Structure

A playbook consists of several key components:

1. **Name**: A descriptive title (e.g., "SaaS Lead Qualification").
2. **Description**: A brief summary of the playbook's purpose.
3. **Goal**: The ultimate objective the assistant is trying to achieve (e.g., "Determine if the user is a qualified buyer and collect their email address").
4. **Instructions (The Flow)**: The step-by-step guide the assistant must follow to reach the goal.

### Writing Effective Instructions

The instructions are the core of your playbook. They should be clear, sequential, and account for different user responses.

**Best Practices for Playbook Instructions:**
- **Use numbered steps**: Clearly define the order of operations (e.g., "1. Ask for their name. 2. Ask for their company size.").
- **Be specific about data collection**: If you need an email address, explicitly tell the assistant to ask for it and validate it.
- **Define exit conditions**: Tell the assistant what to do when the goal is met (e.g., "Once you have the email and company size, thank them and say a representative will be in touch.").
- **Handle edge cases**: Instruct the assistant on how to respond if the user refuses to answer or asks an unrelated question (e.g., "If the user asks about pricing before providing their email, give a brief overview but steer the conversation back to collecting their contact info.").

## Versioning Playbooks

Playbooks in Autoch.at are version-controlled. This means you can safely make changes to a playbook without immediately affecting live conversations.

When you edit a playbook, you create a new **Draft** version. Once you are satisfied with the changes, you can **Publish** the new version, making it the active playbook for any attached assistants.

You can always view the history of a playbook and revert to a previous version if a new change doesn't perform as expected.

## Attaching Playbooks to Assistants

A playbook does nothing on its own; it must be attached to an Assistant.

1. Go to **AI Assistant > Assistants**.
2. Select the assistant you want to use the playbook.
3. In the assistant's configuration, find the **Playbooks** section.
4. Select the playbook(s) you want to attach.

<Callout kind="info">
An assistant can have multiple playbooks attached. The AI will determine which playbook is most relevant based on the user's initial message and intent.
</Callout>

## Playbook Experiments

Autoch.at allows you to run A/B tests on your playbooks to see which version performs better.

Navigate to **AI Assistant > Experiments** to set up a test. You can compare two different versions of a playbook (e.g., one that asks for an email first vs. one that asks for company size first) and measure which one results in a higher qualification rate.

See the [Experiments](/pages/ai-config/experiments) documentation for more details on setting up and analyzing A/B tests.
