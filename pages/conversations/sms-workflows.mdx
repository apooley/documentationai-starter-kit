## What are SMS Workflows?

While Email Workflows handle asynchronous communication, **SMS Workflows** (located at **Conversations > SMS Workflows**) are designed for the immediate, high-engagement nature of text messaging.

SMS workflows allow you to automate sequences of text messages, manage opt-ins and opt-outs, and handle routing based on the content of an incoming text or the status of a lead.

## Key Concepts

To effectively use SMS Workflows, you need to understand a few core concepts:

### Triggers

A workflow is initiated by a **Trigger**. This is the event that causes the workflow to start.

Common triggers include:
- **New SMS Received**: A text message arrives at a specific provisioned phone number.
- **Keyword Received**: A user texts a specific word (e.g., "JOIN", "STOP", "HELP").
- **Tag Added**: A specific tag is applied to a conversation (e.g., "SMS Lead").
- **Lead Status Changed**: A contact's status in your CRM changes (e.g., from "New" to "Qualified").

### Conditions

Once a workflow is triggered, it evaluates **Conditions** to determine if it should proceed.

Conditions act as filters:
- **Phone Number**: Only run the workflow if the text comes from a specific area code or country.
- **Message Contains**: Only run if the message includes certain keywords (e.g., "Appointment", "Confirm").
- **Business Hours**: Only run the workflow during (or outside of) your defined business hours.

### Actions

If the conditions are met, the workflow executes one or more **Actions**.

Actions are the steps the workflow takes:
- **Send SMS**: Send a pre-written text message template or an AI-generated response.
- **Add Tag**: Apply a tag to the conversation for organization.
- **Assign to Team/Agent**: Route the text to a specific human or group.
- **Update CRM**: Change a field in the customer's CRM record.
- **Wait**: Pause the workflow for a specified duration before executing the next action.

## Creating an SMS Workflow

1. Navigate to **Conversations > SMS Workflows**.
2. Click **Create Workflow**.
3. Give your workflow a descriptive name (e.g., "Appointment Reminder" or "Opt-in Confirmation").
4. Define the **Trigger** that will start the workflow.
5. Add any necessary **Conditions** to filter when the workflow runs.
6. Add the **Actions** you want the workflow to perform.
7. Save and activate the workflow.

## Common Use Cases

### 1. The Opt-in Confirmation

This workflow ensures compliance by confirming a user's intent to receive text messages.

- **Trigger**: Keyword Received ("JOIN").
- **Action 1**: Send SMS (Template: "Welcome to Autoch.at alerts! Reply STOP to cancel at any time.").
- **Action 2**: Update CRM (Set "SMS Opt-in" to True).

### 2. The Appointment Reminder

This workflow automates reminders for scheduled events.

- **Trigger**: Tag Added ("Appointment Scheduled").
- **Action 1**: Wait (24 hours before appointment).
- **Action 2**: Send SMS (Template: "Reminder: You have an appointment tomorrow at [Time]. Reply CONFIRM or CANCEL.").
- **Condition**: Has the user replied? (If Yes, exit workflow).
- **Action 3**: Wait (2 hours before appointment).
- **Action 4**: Send SMS (Template: "Your appointment is in 2 hours. See you soon!").

### 3. The Keyword Auto-Responder

This workflow provides immediate answers to common questions via text.

- **Trigger**: Keyword Received ("HOURS").
- **Action 1**: Send SMS (Template: "Our business hours are Monday-Friday, 9 AM to 5 PM EST.").

## Managing Workflows

As your processes evolve, you can easily update your workflows.

- **Edit**: Modify the triggers, conditions, or actions of an existing workflow.
- **Pause/Activate**: Temporarily disable a workflow without deleting it.
- **Duplicate**: Create a copy of a workflow to use as a starting point for a new one.
- **Delete**: Permanently remove a workflow that is no longer needed.
